# v1.5.1 - ADMIN PANEL OVERHAUL

## What's new?

- First time you boot the container, the site is locked until you go through the admin panel setup
- A random password gets auto-generated and printed to your container logs on first boot so you can actually get in
- Added a "Forgot password?" button on the admin login — it prints a one-time recovery code to your logs so you can get back in without restarting anything
- All your settings (Jellyfin, Discord, yt-dlp etc.) can now be configured entirely through the admin panel, no .env file needed
- If you do have a .env, it still works as a fallback — it'll just print a warning in the logs saying it's using it
- The .env file is now optional entirely, server starts fine without one
- Fixed a security issue where YouTube download URLs weren't being validated (could have been used to make the server hit internal addresses)
- Fixed the YouTube download endpoint not requiring auth
- Few video player bug fixes — duration resets properly when switching streams, reconnect messages now show in the buffering spinner
