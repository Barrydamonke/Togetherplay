# v1.4.10 - Bug fixes

## What's fixed?

- **Sync keep-up wasn't working** — The app is supposed to nudge viewers back in sync every 5 seconds, but it wasn't actually firing in busy rooms. This is now fixed.
- **Tap to sync didn't unpause** — If your video was paused when you tapped the sync indicator, it would jump to the right spot but stay paused. It now resumes playing as expected.
- **Suggestions couldn't be retried after an error** — If sending a suggestion failed (e.g. a network blip), the Send button stayed greyed out with no way to try again without closing the popup. You can now retry straight away.
- **Suggestions could be spammed** — There was no limit on how many suggestions could be sent, which could have been used to flood the notifications. A limit of 5 per minute is now in place.
- **Suggestion type wasn't validated** — The suggestion type (media/feature/bug) wasn't checked on the server, so a malformed request could inject arbitrary text into the Discord notification. Only valid types are now accepted.
- **Playback position wasn't validated** — Timestamps sent from clients weren't checked for invalid values like NaN or negative numbers, which could have broken playback for everyone in the room. These are now rejected.
- **Closed captions button was unresponsive** - It should now tell you if there are no subtitiles instead of just failing, including a button to request new ones.
